[2] BROWNBOO — POND-BOTTOM GROUND-GRID CAPTURE  (the "other brownboo thing")
---------------------------------------------------------------------------
On spot load, IF DC_DUMP_DIR is set (see .env.sample), the mod dumps the base-ground grid to
$DC_DUMP_DIR/ground_grid.csv (the accurate pond bottom, including the underwater cells collision skips).
If DC_DUMP_DIR is unset the dump is skipped entirely.

  - Check the LOG: "GROUND GRID: dumped N cells across M area(s)".
      If it says "bad dims" for an area -> tell Claude the numbers (grid layout differs from the assumed
      bounds; the dump code needs adjusting).
  - Then re-run:  python3 tools/brownboo_viewer.py
      Toggle "ground grid: pond BOTTOM" and confirm it forms the bowl shape (not flat / not garbage).


[3] PRISCLEEN (DC2 fish) — NOW WIRED, EXPERIMENTAL/UNTESTED  (do this AFTER [1] and [2] work)
------------------------------------------------------------------------------------------
The DC2 Priscleen model is extracted, converted to a DC1-format .chr at game_data/priscleen/f19a.chr
(untracked game data; the .csproj copies it to Resources/Fish/ at build), and the runtime injection is
wired (PriscleenFish.cs). It injects into DC1's cut species 8 when a species-8 fish is caught in Brownboo.

NOTE: the fish-depth override was REMOVED (fish are back at vanilla WaterLevel-12) — the earlier -6 depth
left the fish shallower than the hook reached, so nothing could bite. Fish should bite normally now.

Species 8 doesn't spawn normally, so to SEE Priscleen you must flip the test switch:
  In PriscleenFish.cs set  ForceAllSpecies8 = true   (stamps every Brownboo fish as species 8), rebuild.
  Then fish in Brownboo and CATCH ANY FISH -> it should reel up as the Priscleen model, flapping.

FIXED (was showing "Bobo" = the f01a stand-in): the swap used to match the BG slot by a HARD-CODED size
and only the first match. It now matches by FILENAME (f01a) across ALL 32 BG slots, gated on the caught
species (BattleFish == 8) — robust to whichever BG/fish slot is used. There's a ~120-frame window after the
read completes, so no race.

Watch the LOG:
  - "[PriscleenFish] name_419[8] redirected to stand-in f01a ... Priscleen model 131536B ready" (spot load)
  - "[PriscleenFish] TEST: stamped N Brownboo fish as species 8"                          (on fishing start)
  - "[PriscleenFish] injected Priscleen (131536B) into BG slot N buffer 0x........ (file 'chara/f01a.chr')"
                                                                                          (when you reel one in)
If the 3rd line never prints, tell Claude the log — the gate (species) or filename match needs adjusting.

What to look for / report (these are the untested unknowns):
  - Does the caught fish show the Priscleen MODEL at all (vs invisible / garbage / crash)?
  - Does it FLAP (motion index 1 = the reeled-in animation)?  [KEYs kept in DC2 order 0=4-14/1=18-28/2=40-80]
  - Texture correct?  (VERTEX_ANIME 1 vs DC2's skin1/skin2 — may need an ALLOC_DBUFF tweak)
  - Any crash on catch?  (if so, note the exact moment)

NOTE: this test aid makes ALL Brownboo fish Priscleen and doesn't set the name/stats yet. The real
feature (species 8 in the spawn table + fish_info[8] + "Priscleen" name via .mes patch) comes after the
model itself is confirmed working. Set ForceAllSpecies8 back to false for normal play.
